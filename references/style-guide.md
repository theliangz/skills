# 大模型视界文章风格指南

## 一、核心写作原则

### 1. 专业性与可读性平衡

- 技术术语首次出现时，可中英对照：`思维树（Tree-of-Thoughts，ToT）`
- 复杂概念使用类比/比喻解释
- 避免过度学术化，保持工程师视角

### 2. 数据驱动

- 性能 claims 必须有数据支撑
- 与基线/竞品对比说明优势
- 引用权威来源（论文、官方报告）

### 3. 实战导向

- 提供可运行的代码示例
- 包含架构图/流程图
- 给出生产化部署建议

## 二、格式规范

### 关键词强调

使用双下划线加粗（微信公众号 Markdown 格式）：

```
__原生多模态__
__Agent Swarm__
__联合优化__
```

### 章节编号

两种可选格式：

```
### 01 原生多模态：打破"后期融合"的魔咒
### 02 Zero-Vision SFT：用纯文本激活视觉潜能
### 03 Agent Swarm：从"串行"到"蜂群"
```

或：

```
### 一、规模与效率的双重飞跃：744B 参数与 DSA 架构
### 二、强化学习的新基建：slime 异步架构
### 三、进击的智能体：在复杂环境中"经营"与"执行"
```

### 图片说明

使用斜体格式：

```
![架构图](图片链接)
_Agent Swarm 架构。可训练的编排者（Orchestrator）动态创建专门的冻结子智能体，并行执行子任务。_
```

### 代码块

指定语言类型：

````python
from pydantic import BaseModel, Field

class PuzzleState(BaseModel):
    """表示过河问题的状态"""
    left_bank: set[str] = Field(default_factory=lambda: {"wolf", "goat", "cabbage"})
````

````python
# LLM 客户端核心逻辑
class LLMClient:
    def __init__(self, config=None):
        # 支持多模型切换与故障转移
        self.config = config or get_settings().llm
````

## 三、各系列专项规范

### 智能开发实战系列

**目标读者**：有一定基础的 Agent 开发者

**重点内容**：
- 架构设计思路
- 核心代码实现（可直接使用）
- 框架/工具选型理由
- 常见坑点与解决方案

**代码量**：核心代码片段 3-5 处，每处 10-30 行

### AI快讯分享系列

**目标读者**：关注前沿动态的技术人

**重点内容**：
- 一句话概括核心亮点
- 与前代/竞品对比
- 性能基准测试数据
- 对开发者的实际影响

**文章长度**：800-1200 字

### 优秀论文分享系列

**目标读者**：希望深入理解论文的工程师

**重点内容**：
- 论文解决的问题
- 核心方法/架构
- 关键实验结果
- 与 SOTA 对比
- 可落地的启发

**配图要求**：至少 2 张论文原图

### 开源项目实战系列

**目标读者**：希望从零搭建系统的开发者

**重点内容**：
- 完整架构图
- 关键模块代码
- 部署配置
- 生产化建议（性能、安全、监控）

**项目要求**：提供可运行的 GitHub 链接

## 四、开篇与结尾模板

### 开篇模板

```
大家好，我是视界君。

在通往【技术领域】的道路上，【公司/团队】再次交出了一份令人瞩目的答卷。

昨日，【来源】正式发布了 __【技术名称】__ 。作为一个【定位】，它不仅【核心能力1】，更在【核心能力2】上取得了突破性进展。

我通读了这份【报告/论文】后，提炼了【技术名称】的核心技术亮点，供大家参考。
```

### 结尾模板

```
## 总结

【技术名称】的发布，不仅是【技术层面的升级】，更是对【核心概念】的一次深度思考。

通过【核心价值】，【团队/作者】正在加速社区向【目标】迈进。无论是【亮点1】带来的【价值1】，还是【亮点2】带来的【价值2】，都值得每一位【领域】从业者深究。
```

## 五、常见词汇库

### 大模型相关

- 通用智能体（General Agentic Intelligence）
- 原生多模态（Native Multimodal）
- 联合预训练（Joint Pre-training）
- 后训练（Post-training）
- 强化学习（Reinforcement Learning, RL）
- 思维链（Chain-of-Thought, CoT）
- 思维树（Tree-of-Thoughts, ToT）

### Agent 相关

- 智能体蜂群（Agent Swarm）
- 编排者（Orchestrator）
- 子智能体（Sub-agent）
- 工具调用（Tool Calling）
- 规划（Planning）
- 执行（Execution）
- 反思（Reflection）

### 工程实践

- 检索增强（Retrieval-Augmented Generation, RAG）
- 向量数据库（Vector Database）
- 语义检索（Semantic Search）
- 自愈闭环（Self-healing Loop）
- 生产化（Production-ready）
